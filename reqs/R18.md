# R18 — Documentation module (info.md + docs/)

**Requirement.** The system shall provide a documentation module that (a) maintains
an `info.md` holding explanatory project documentation and auto-loaded alongside
`CLAUDE.md`, (b) on apply outsources explanatory documentation out of `CLAUDE.md`
into `info.md`, (c) consolidates scattered reference material (API references,
manuals, specs) under a `docs/` folder linked from `info.md`, and (d) installs a
`CLAUDE.md` rule that documentation belongs in `info.md`/`docs/`, not in `CLAUDE.md`
— all losslessly.

## Rationale
In many projects a large part of `CLAUDE.md` is really documentation of how the code
works. That bloats the always-loaded rules file and mixes two concerns. Splitting it
out into a dedicated `info.md` (still auto-loaded, via Claude Code's `@import`) keeps
`CLAUDE.md` to rules while giving documentation a durable home; bulky reference
material moves to `docs/`, read on demand. Traces to **G6** (and serves G4 — docs as
durable memory).

## Acceptance criteria
Verifiable conditions; all must hold before R18 is marked done in `reqs.md`.

- [x] `LLMbootstrap.md` defines a Documentation module (id `docs`) with Structure +
      align and Install phases, listed in **Modules** and the module conventions.
- [x] Install creates `info.md` (if absent) and a managed `CLAUDE.md` block that
      imports it (`@info.md`) so it auto-loads, plus the rule to keep documentation
      in `info.md`/`docs/`, not `CLAUDE.md`.
- [x] On apply to an existing project, explanatory documentation in `CLAUDE.md` is
      outsourced into `info.md` losslessly; workflow rules and managed blocks stay.
- [x] Scattered reference material (API refs, manuals, specs, stray doc `*.md`) is
      consolidated under `docs/`, linked from `info.md`, content preserved; `reqs.md`,
      `README.md`, and `LLMbootstrap.md` are never absorbed into `docs/`.
- [x] All documentation moves pass through the R17 confirmation gate and respect R16
      (lossless) and R6 (no fabrication).

## Design notes
- Auto-load mechanism: a line `@info.md` inside the module's managed `CLAUDE.md`
  block (Claude Code imports `@`-referenced files into context).
- Boundary vs. Module 2: `README.md` is the short *outward* overview derived from
  goals; `info.md`/`docs/` are the detailed *internal* documentation.
- `docs/` is created only when there is reference material to hold.
