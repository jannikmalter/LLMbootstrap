# B1 — Confirm-before-writing has no pre-authorization escape

## Summary
The Apply procedure (step 3) and R4 mandate presenting the proposed structured
result and getting user confirmation *before writing*. When the user gives a
direct, pre-authorized instruction ("apply LLMbootstrap.md, infer from chat"),
following the mandate forces either an unwanted extra round-trip or — as happened
on the first self-apply — writing first and confirming after, i.e. violating the
procedure. Violates **R4**.

## Steps to reproduce
1. Tell the agent "apply LLMbootstrap.md" with no further interaction expected.
2. Project has no `reqs.md`, so the Structure phase triggers.
3. The procedure requires a confirm step the user did not ask for.

## Expected
A direct "apply" proceeds without a confirmation prompt; confirmation is required
only when the user hasn't pre-authorized.

## Actual
The procedure unconditionally requires confirmation, contradicting the direct
instruction.

## Notes / cause
The file models confirmation as mandatory and has no notion of pre-authorization
or a "proceed without asking" signal.

## Fix
- [x] Reword step 3 / R4 to "confirm unless the user has authorized proceeding."
- [x] Define what counts as pre-authorization (explicit "apply", "don't ask", etc.).
- [x] Keep the default (interactive) confirm for ambiguous / unattended runs.

Resolved in `LLMbootstrap.md` Apply procedure step 3.
