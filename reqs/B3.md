# B3 — Safety rules are not installed where they operate

## Summary
The safety rules — "never modify `LLMbootstrap.md`", "only managed blocks are
owned by this tool", "don't fabricate" — live in `LLMbootstrap.md`. But after an
apply, the agent works day-to-day from `CLAUDE.md` and does not re-read the source.
So in a normal future session (especially in this meta-project, which contains the
source), nothing tells the agent that `LLMbootstrap.md` is special. Violates **R5**.

## Steps to reproduce
1. Apply the file to a project.
2. Start a fresh session; the agent loads `CLAUDE.md` only.
3. Nothing in `CLAUDE.md` protects `LLMbootstrap.md` or explains block ownership.

## Expected
The persistent rules that must govern ongoing work are present in `CLAUDE.md`.

## Actual
Those rules exist only in the un-loaded source file.

## Notes / cause
Rules that govern *application* and rules that govern *ongoing work* were not
separated. The latter must be installed as a managed block.

## Fix
- [x] Split safety rules: application-time (stay in `LLMbootstrap.md`) vs.
      ongoing (install into `CLAUDE.md`).
- [x] Add a small managed `CLAUDE.md` block (e.g. module=core) carrying the
      ongoing rules, so self-application protects `LLMbootstrap.md`.

Resolved: added **Module 0 — Core conventions** (`module=core`) and split the
Safety rules section in `LLMbootstrap.md`. Installed into this project's `CLAUDE.md`.
