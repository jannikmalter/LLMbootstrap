# R17 — Confirmation summary before writing

**Requirement.** Apply shall, before writing any change, gather all findings and
present the user a short confirmation summary of the proposed changes — what will
be created, restructured, and installed, with the source of each inferred item and
any open questions — and proceed to write only after the user confirms, unless the
user has explicitly opted out of the gate ("don't ask", "just do it").

## Rationale
A restructure touches the project's durable memory (G4). The user wants to see what
an apply intends to do — what it inferred and from where — and approve it before any
file is changed, rather than discovering the result after the fact. Making the
confirmation summary the default keeps applies predictable and correctable. This
revises the default established by **B1**: a plain "apply" no longer writes directly;
it confirms first. The explicit opt-out preserved from B1 still lets a user skip the
gate when they want an unattended run. Traces to **G2** (safe, repeatable operation).

## Acceptance criteria
Verifiable conditions; all must hold before R17 is marked done in `reqs.md`.

- [x] The Apply procedure separates planning (gather + compute changes, no writes)
      from execution (writes), with a confirmation gate between them.
- [x] The gate presents a *short* summary — per module, what will be created /
      restructured / installed, the source of inferred items, and open questions —
      not a full file dump; the full diff is offered on request.
- [x] No project file is written until the user confirms, except under an explicit
      opt-out, which proceeds directly and still reports what was done.
- [x] A plain "apply" / "execute" instruction triggers the gate by default (it is
      no longer treated as pre-authorization to write immediately).
- [x] Files already matching the conventions are reported as unchanged and not
      rewritten (convergent — consistent with R7).

## Open questions
- Whether to keep the explicit opt-out at all, or make the gate unconditional.
  Current decision: keep the opt-out (default = confirm). Revisit if the user wants
  the gate to be mandatory with no escape.
