# R4 — Structure an existing project's scattered information

**Requirement.** On the first apply to an existing project, the system shall sweep
the project for information that already exists — however scattered — and
reorganize it into this file's conventions, confirming the result with the user
before writing.

## Rationale
Applying to a messy existing project is the main event: it ingests the mess once,
then keeps it tidy via the installed `CLAUDE.md` rules. This is migration of what's
already there, not invention. Traces to **G1**.

## Acceptance criteria
Verifiable conditions; all must hold before R4 is marked done in `reqs.md`.

- [x] The sweep covers README/docs, an existing `CLAUDE.md`, `TODO`/`FIXME`/`HACK`
      comments, the issue tracker, and scattered `*.md` files.
- [x] Findings are mapped into the module's format (goals, requirements, bugs, todos).
- [x] The proposed result is presented with the source of each item and confirmed
      before writing.
- [x] Implied-but-ambiguous items become open questions rather than table rows.
- [x] Relocating/deduping/reformatting explicit info is done; fabrication is not.

## Idempotency
- [x] Runs only when the working file (`reqs.md`) is absent or empty.
- [x] Once populated, the working file is user-owned and not re-scanned unless the
      user explicitly asks to re-ingest. (R7)
