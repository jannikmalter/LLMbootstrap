# R3 — Managed blocks with id+version markers

**Requirement.** Every block this file writes into `CLAUDE.md` (or any shared
file) shall be wrapped in markers carrying the module id and a version, so that
re-applying can insert, replace, or leave the block untouched without duplicating.

## Rationale
"Apply" must reconfigure an existing project to an updated workflow. Blind-append
would duplicate blocks on every run. Markers make the operation idempotent and let
a version bump signal that a block is stale. Traces to **G2**.

## Acceptance criteria
Verifiable conditions; all must hold before R3 is marked done in `reqs.md`.

- [x] Each managed block is delimited by `START`/`END` markers naming `module=<id>`.
- [x] A block carries a version (`v=<n>`).
- [x] Apply inserts the block when no `START` marker for that id exists.
- [x] Apply replaces marker-to-marker content when the id matches but the version differs.
- [x] Apply leaves the block untouched when id and version both match.
- [x] Content outside any marker (the user's own prose) is never altered.

## Open questions
- The version currently appears in both the module heading and the marker, kept in
  sync by hand. Should the marker be the single source of truth instead?
